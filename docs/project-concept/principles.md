# Принцип работы

## 1. Сначала выбирается событие, а не текст

Система сначала ищет подходящие события на нужную дату в календарях релизов.
Это значит, что в основе прогноза лежит **не фантазия модели**, а уже запланированный инфоповод.

## 2. Сначала строится сценарий, потом пишется новость

Проект работает в два шага:

1. строит несколько фактических сценариев;
2. только потом пишет заголовок и первый абзац.

Такой подход нужен, чтобы модель не "сочиняла красивую новость", а опиралась на ограниченный и правдоподобный набор вариантов.

## 3. Прогноз делается под конкретное издание

Проект не пишет "новость вообще".
Он пытается имитировать подачу выбранного источника, используя локальный архив похожих материалов как стилевые ориентиры.

## 4. Побеждает не самый яркий вариант, а самый устойчивый

Внутри проекта предпочтение отдаётся вариантам, которые:

- основаны на событии с высокой вероятностью публикации;
- похожи на привычную манеру выбранного издания;
- не содержат лишней фантазии;
- не перегружены неподтверждёнными деталями.

## 5. Почему это практично

Такой принцип работы делает систему полезной не только для конкурса, но и для любого сценария, где нужно аккуратно прогнозировать медийную реакцию на запланированные события:

- макроэкономические релизы;
- бизнес-опросы;
- корпоративные анонсы;
- регулярные статистические публикации.

Итоговая мысль простая:
проект не "угадывает будущее", а **системно прогнозирует, как медиа оформят заранее ожидаемый повод**.
